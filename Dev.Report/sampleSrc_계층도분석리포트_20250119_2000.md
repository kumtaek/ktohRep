## 🔍 아키텍처 분석

### 🏛️ **계층형 아키텍처 (Layered Architecture)**

```
┌─────────────────────────────────────┐
│           Presentation Layer        │
│         (Controller 패키지)         │
├─────────────────────────────────────┤
│           Business Layer            │
│          (Service 패키지)           │
├─────────────────────────────────────┤
│           Data Access Layer        │
│          (Mapper 패키지)            │
├─────────────────────────────────────┤
│           Integration Layer        │
│        (Integration 패키지)         │
├─────────────────────────────────────┤
│           Configuration Layer      │
│          (Config 패키지)            │
└─────────────────────────────────────┘
```

### 🔄 **데이터 흐름**

```
사용자 요청 → Controller → Service → Mapper → Database
     ↑                                 ↓
     └────────── Response ←────────────┘
```
